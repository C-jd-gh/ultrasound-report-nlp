# 超声报告 NLP 小型系统项目说明

## 1. 项目简介

本项目是一个面向自然语言处理课程任务的超声诊断报告 NLP 小型网页系统。系统围绕甲状腺、乳腺、肝胆胰脾三类超声报告文本，完成从数据预处理、医学词典构建、分词、关键词抽取、正则匹配、实体识别、文本规范化，到模板报告生成和相似报告检索的一整套流程。

项目没有引入 BERT 或 TensorFlow 深度学习训练，而是采用 **规则方法 + 传统机器学习方法**。系统同时覆盖 jieba 分词、词性标注、TF-IDF、TextRank、CRF、Word2Vec、Logistic Regression 和 KMeans 等课程常见技术，优点是实现过程清楚、结果可解释，适合课程展示和报告讲解。

## 2. 项目目标

本项目主要想解决的问题是：

- 如何对超声诊断报告进行文本预处理
- 如何根据医学术语构建专用词典
- 如何对医学报告进行较合理的分词
- 如何从报告中抽取关键词和医学实体
- 如何用正则表达式抽取尺寸、部位、回声、边界、血流等结构化信息
- 如何把非结构化报告文本转成结构化摘要
- 如何根据结构化结果生成模板化报告
- 如何在网页中直观展示 NLP 处理流程

## 3. 数据集说明

项目使用的数据位于 `USData/` 目录，包含三类超声报告：


数据规模如下：

| 报告类型 | 总数 | 训练集 | 测试集 | 验证集 |

## 4. 我们做了什么

### 4.1 数据预处理

项目新增了独立的数据预处理脚本：

```text
scripts/preprocess_data.py
```

数据预处理主要完成以下工作：

- 读取原始 JSON 数据
- 兼容 UTF-8 BOM 编码
- 修复部分文本编码异常
- 去除多余空白字符
- 保留医学占位符
- 统计每类报告中的占位符数量
- 为每条报告生成分词预览
- 生成清洗后的数据文件
- 生成数据统计文件
- 生成医学词典文件

预处理后会生成：

```text
processed_data/
  thyroid_clean.json
  mammary_clean.json
  liver_clean.json
  dataset_stats.json
  medical_dictionary.json
  preprocess_summary.json
```

原始数据 `USData/` 不会被修改，系统优先读取 `processed_data/` 中的清洗结果。如果清洗数据不存在，则回退读取原始数据。

### 4.2 医学词典构建

词典采用半自动方式构建，主要有三个来源：

1. 原始技术词表：来自 `USData/key_technical_words.txt`
2. 人工整理的医学分类词典
3. 从报告文本中统计得到的高频医学短语

人工整理的词典按照类别划分：

| 类别 | 示例 |
| --- | --- |
| 器官词 | 甲状腺、乳腺、肝脏、胆囊、胰腺、脾脏 |
| 部位词 | 左叶、右叶、峡部、腋下、双侧颈部、原区域 |
| 病灶词 | 结节、低回声结节、占位性病变、钙化、肿大淋巴结 |
| 回声词 | 低回声、高回声、无回声、囊实混合回声 |
| 边界词 | 边界清晰、边界欠清晰、包膜光滑 |
| 形态词 | 形态规整、形态欠规整、类圆形、分叶状 |
| 血流词 | CDFI、血流信号、未见异常血流信号 |
| 状态词 | 术后、全切术后、未见明确占位性病变 |
| 测量词 | `_2DS_`、`_3DS_`、`_SCM_` |

词典主要用于分词、实体识别、关键词统计和规则解释。

### 4.3 分词与词性标注

本项目升级后使用 `jieba` 作为基础中文分词工具，并结合超声报告医学自定义词典进行领域适配。系统会把医学词典中的专用术语加入 jieba，例如 `低回声结节`、`甲状腺全切术后`、`未见明确占位性病变`、`双侧颈部` 等。`jieba` 是项目必需依赖，不再使用无库兜底方案。

分词流程如下：

```text
报告文本 -> 清洗文本 -> 加载 jieba -> 导入医学自定义词典 -> jieba 分词 -> 过滤无意义单字 -> 输出分词结果
```

例如：

```text
甲状腺全切术后，原区域未见明确占位性病变。双侧颈部未见异常肿大淋巴结。
```

可以分为：

```text
甲状腺全切术后 / 原区域 / 未见明确占位性病变 / 双侧颈部 / 未见异常肿大淋巴结
```

这样能够保留医学报告中的关键短语，而不是简单地拆成单字。

词性标注使用 `jieba.posseg` 完成，页面会显示每个词的词性、词性名称，以及它是否命中医学词典。词性结果可用于过滤名词、专有名词和医学术语，为关键词抽取和实体识别提供辅助。

### 4.4 关键词抽取

系统实现了三种关键词抽取方式：

1. 词频关键词
2. TF-IDF 关键词
3. TextRank 关键词

词频关键词用于查看当前报告中出现最多的词；TF-IDF 关键词用于突出当前报告中相对更有区分度的医学词；TextRank 关键词基于词共现图和 PageRank 思想，可以对单份报告进行无监督关键词抽取。

例如一条报告中可能抽取出：

```text
甲状腺
全切术后
未见明确占位性病变
未见异常肿大淋巴结
```

这些关键词可以帮助快速理解报告主题。

### 4.5 正则匹配

系统使用正则表达式抽取报告中的结构化信息，包括：

- 测量信息
- 部位信息
- 血流信息
- 边界信息
- 形态信息
- 回声信息
- 淋巴结信息
- 术后信息
- 病灶信息

例如：

```text
左叶中部可见一低回声结节，大小约_2DS_，边界清晰，CDFI示未探及血流信号。
```

可以抽取出：

| 类型 | 内容 |
| --- | --- |
| 部位 | 左叶中部 |
| 回声 | 低回声结节 |
| 测量 | `_2DS_` |
| 边界 | 边界清晰 |
| 血流 | CDFI示未探及血流信号 |

### 4.6 实体识别

实体识别采用两条路线并行展示：

1. 词典匹配和正则规则实体识别
2. 弱标注 BIO 数据训练的 CRF 序列实体识别

系统识别的实体类型包括：

- 器官
- 部位
- 病灶
- 回声
- 边界
- 形态
- 血流
- 测量
- 淋巴结
- 术后状态

实体识别结果会显示实体类型、实体名称、实体值和来源。来源可能是：

- `dictionary`：来自医学词典
- `regex`：来自正则规则
- `sentence_rule`：来自句子级病灶规则

这样可以解释每个实体是怎么被系统识别出来的。

CRF 模型训练脚本为：

```text
scripts/train_ner.py
```

训练思路是先用现有词典和正则规则自动生成字符级 BIO 弱标注，再用 `sklearn-crfsuite` 学习上下文特征。模型按报告类型分别训练，训练后生成：

```text
models/
  thyroid_sequence_ner.pkl
  mammary_sequence_ner.pkl
  liver_sequence_ner.pkl
  ner_metrics.json
```

网页会展示 CRF 识别出的实体、实体类型、位置和来源。

### 4.7 病灶-器官关系抽取

系统新增了简易句法/关系抽取模块，不构造完整句法树，而是基于标点切句、触发词和位置规则抽取病灶所属关系。例如：

```text
甲状腺右叶见低回声结节，大小约_2DS_，CDFI示未探及血流信号。
```

可以抽取为：

```text
甲状腺右叶 -> 结节 -> _2DS_ -> CDFI示未探及血流信号
```

该模块用于展示“主语/部位 - 触发词 - 病灶 - 属性”的结构化关系。

### 4.8 文本规范化

医学报告中存在很多同义表达，例如：

| 原表达 | 规范化表达 |
| --- | --- |
| 边界清楚 | 边界清晰 |
| 形态规则 | 形态规整 |
| 未探及 | 未见 |
| 无明显 | 未见明显 |
| 不扩张 | 未见扩张 |

文本规范化的作用是减少同义表达带来的差异，让后续规则抽取和模板生成更稳定。

### 4.9 模板报告生成

系统会根据实体识别和正则抽取结果，生成结构化摘要。

例如：

```text
甲状腺超声结构化摘要：
共识别到 1 条病灶相关描述。
1. 左叶中部，低回声结节，_2DS_，边界清晰，形态规整，CDFI示未探及血流信号。
以上为模板化文本摘要，仅用于 NLP 课程演示。
```

模板生成不是医学诊断，而是对原报告的结构化复述。

### 4.10 报告质量检查

系统会根据报告内容进行简单质量检查。

例如，如果发现报告中有病灶描述，但缺少尺寸信息，就会提示：

```text
发现病灶描述，但缺少尺寸信息。
```

目前检查的内容包括：

- 是否缺少尺寸信息
- 是否缺少部位信息
- 是否缺少 CDFI 或血流信息
- 是否缺少边界信息
- 是否缺少形态信息

### 4.11 相似报告检索

系统使用 TF-IDF 向量计算报告之间的文本相似度。

用户输入或选择一条报告后，系统可以从同类报告中找出最相似的几条报告，并显示：

- 报告 UID
- 数据划分
- 标签
- 相似度分数
- 报告文本

这个功能可以用于病例检索和相似文本分析。

### 4.12 标签分类预测

为了进一步提升项目技术含量，系统新增了传统机器学习分类模块。

分类模型使用：

```text
TF-IDF 特征 + Logistic Regression 分类器
```

训练脚本为：

```text
scripts/train_classifier.py
```

模型训练方式：

- 每个报告类型单独训练一个分类器
- 使用 `train` 数据训练
- 使用 `test` 数据作为主要测试集
- 使用当前医学分词结果作为 TF-IDF 的分词输入
- 预测目标为数据集中的 `labels` 字段

训练完成后生成：

```text
models/
  thyroid_classifier.pkl
  mammary_classifier.pkl
  liver_classifier.pkl
  classifier_metrics.json
```

当前测试集效果：

| 报告类型 | Accuracy | Macro F1 |
| --- | ---: | ---: |
| 甲状腺 | 0.9004 | 0.9235 |
| 乳腺 | 0.9147 | 0.9053 |
| 肝胆胰脾 | 0.9247 | 0.8937 |

网页中会展示预测标签、置信度、Top-3 标签概率、准确率、F1 和混淆矩阵。该模型用于课程项目中的文本分类演示，不作为医学诊断依据。

### 4.13 Word2Vec 向量化与 KMeans 聚类

系统新增向量训练脚本：

```text
scripts/train_vectors.py
```

该脚本完成两类任务：

1. 使用 `gensim` 训练超声报告领域 Word2Vec 词向量
2. 使用 `TF-IDF + KMeans` 对同类报告进行聚类

训练后生成：

```text
models/
  ultrasound_word2vec.model
  thyroid_kmeans.pkl
  mammary_kmeans.pkl
  liver_kmeans.pkl
  vector_metrics.json
```

Word2Vec 可用于查询相近医学术语，KMeans 可用于把报告按文本特征自动分簇，并展示每个簇的高频词和代表报告。

### 4.14 良恶性倾向提示

本项目没有训练 LSTM 良恶性分类模型，而是把“情感分析”的正负词典思想迁移到超声报告场景。系统维护两类词：

- 可疑征象词：形态不规则、边界欠清晰、微小钙化、纵横比、血流丰富等
- 良性/正常表达：边界清晰、形态规整、未见血流信号、未见明显占位等

系统根据命中词计算倾向分，输出“良性/常规倾向、需关注倾向、可疑倾向”。该结果只用于 NLP 课程演示，不作为医学诊断。

## 6. 后端 API

系统使用 Flask 搭建本地 Web 后端。Flask 负责提供网页入口、静态文件访问和 JSON API。前端页面通过 JavaScript 的 `fetch` 调用这些接口，再把 NLP 分析结果渲染到页面上。

主要接口包括：

| 接口 | 功能 |
| --- | --- |
| `GET /api/report/sample` | 获取一条样本报告 |
| `POST /api/analyze` | 分析一条报告文本 |
| `POST /api/similar` | 检索相似报告 |
| `POST /api/cluster` | 预测报告所属聚类簇 |
| `POST /api/vector/nearest` | 查询 Word2Vec 相近词 |
| `GET /api/stats` | 获取数据集统计 |
| `GET /api/dictionary` | 获取词典内容 |
| `GET /api/organs` | 获取报告类型 |
| `GET /api/model/metrics` | 获取分类模型评估 |
| `GET /api/ner/metrics` | 获取 CRF 实体模型评估 |
| `GET /api/vector/metrics` | 获取向量与聚类模型评估 |

## 9. 项目特点

本项目的特点是：

- 使用真实超声报告文本作为处理对象
- 有独立的数据预处理流程
- 有医学词典构建过程
- 分词、关键词、正则、实体识别、模板生成流程完整
- 结果可解释，适合课程汇报
- 提供网页界面，展示效果直观
- 不依赖深度学习训练，环境要求较低

## 10. 项目局限

当前系统仍然有一些局限：

- 分词依赖词典质量，词典不完整时可能出现错误切分
- 实体识别主要依赖规则，无法覆盖所有复杂表达
- CRF 的训练数据来自弱标注，不等同于人工精标医学实体数据
- 良恶性倾向提示来自词典计分，不等同于临床诊断
- 模板生成是结构化复述，不是医学诊断
- 质量检查规则较简单，只能发现部分明显缺项

## 11. 总结

本项目完成了一个面向超声诊断报告的 NLP 小型系统。系统从原始医学文本数据出发，经过数据预处理、词典构建、Jieba 分词、词性标注、关键词抽取、正则匹配、规则实体识别、CRF 序列实体识别、关系抽取、文本规范化、模板生成、分类预测、Word2Vec 向量化和 KMeans 聚类，最终通过网页形式展示完整处理流程。

整体方案强调可解释性和可展示性，适合作为自然语言处理课程项目。项目没有进行深度学习模型训练，而是通过规则、统计学习和传统机器学习方法实现了医学报告文本的基础分析与结构化处理。
