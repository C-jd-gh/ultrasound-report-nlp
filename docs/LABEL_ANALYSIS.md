# 数据集标签含义审计

## 1. 结论

原始数据没有提供官方标签映射表。本文件中的标签名称来自以下统计推断：

- 每个标签的样本数量
- 高频字符片段
- 医学征象命中比例
- 最常见代表报告
- 不同标签之间的文本模板差异

综合结果表明：

> 数据集中的 `labels` 更接近“报告内容模式/模板类别”，并不完全是标准疾病诊断类别。

其中甲状腺标签的医学含义相对清楚；乳腺标签混合了病灶、左右侧术后状态和报告模板；肝胆胰脾标签大量依赖形态、回声程度、句式和模板格式。

因此分类模型只能称为“原始标签分类”或“报告模板分类”，不能称为疾病诊断模型。

肝胆胰脾标签中，多个标签的医学内容非常接近，区别主要是：

- `形态饱满`、`稍饱满`、`大小形态如常`
- `后场声衰减`、`后场回声衰减`、`后场回声稍衰减`

- 是否包含固定测量占位符

这说明标签明显包含报告模板和措辞风格因素。

## 5. 对分类模型的影响

当前 `TF-IDF + LogisticRegression` 分类模型取得较高准确率，但模型可能主要学习到：

- 固定报告句式
- 占位符组合
- 左右侧和术后关键词
- 文本模板格式
- 某些轻微措辞差异

因此高准确率不能解释为医学诊断准确率。

推荐在页面和课程汇报中使用以下名称：

```text
原始标签预测
报告文本模式分类
数据集模板类别预测
```

不推荐使用：

```text
疾病诊断预测
良恶性分类
临床类别预测
```

## 6. 项目建议

1. 保留分类模块，用于展示 TF-IDF 和逻辑回归的监督学习流程。
2. 将分类模块从报告分析主结论中弱化，放在“课程算法实验”位置。
3. 页面增加说明：原数据未提供官方标签含义，预测结果仅对应原始数字标签。
4. 项目主线应放在分词、关键词、实体识别、关系抽取、规范化和模板摘要。
5. 若要做医学含义明确的分类，需要重新人工标注，例如：
   - 正常/异常
   - 有病灶/无病灶
   - 单发/多发结节
   - 术后/非术后
   - 囊性/实性/囊实混合
