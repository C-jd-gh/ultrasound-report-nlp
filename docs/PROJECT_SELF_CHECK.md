# 项目自检与优化建议

## 1. 当前主要不足

### 1.1 分词质量仍依赖医学词典

当前分词使用 `jieba + 医学自定义词典 + 停用词过滤`。对常见医学短语效果较好，例如：

- 低回声结节
- 囊实混合回声结节
- 未见明确占位性病变
- 双侧颈部

但遇到更复杂的组合表达时，仍可能切碎，例如：

- 以囊性为主
- 右叶近峡部
- 周边及内部可探及血流信号

已优化：新增强制医学短语保护，在分词前先保护常见医学短语，再交给 jieba 处理。

后续可继续优化：建立独立 `resources/medical_phrases.txt` 和 `resources/stopwords.txt`，方便持续维护。

## 2. 标签解释不足

数据集只提供数字 `labels`，没有提供标签编号对应的医学含义。

因此分类模型只能说明：

```text
根据报告文本预测原始数字标签
```

不能直接说明：

```text
标签 1 = 某种具体疾病
```

当前处理方式：保留分类模型，但在汇报中明确说明它是文本分类实验，不作为医学诊断解释。

## 3. 可保留但要弱化的功能

### 3.1 Word2Vec

Word2Vec 对课程展示有价值，因为它覆盖词向量技术。但在网页中直接展示模型路径和维度意义不大。

已优化：页面中弱化模型细节，只保留词表大小、向量维度和用途说明。

### 3.2 KMeans 聚类

KMeans 有课程价值，可展示无监督聚类。但原展示只是簇编号和关键词，解释性较弱。

已优化：改成“报告聚类画像”，展示每个簇的自动名称、样本数、关键词和代表报告。

### 3.3 征象倾向提示

原先叫“良恶性倾向提示”，但数据集没有良恶性标签，容易过度表达。

已优化：改名为“征象倾向提示”，只表示 NLP 规则命中的可疑征象多少，不作为诊断。

## 4. 不建议删除的功能

- 分词、词典、停用词：核心基础
- TF-IDF 关键词：课程重点，必须保留
- 正则抽取：医学报告结构化最直观
- 实体识别：项目核心展示点
- 文本规范化：能体现医学同义表达处理
- 模板生成：能体现从 NLP 到报告摘要的应用
- 相似报告检索：TF-IDF 的实际应用
- 分类模型：传统机器学习完整流程

## 5. 可以考虑隐藏或放到统计页的功能

- Word2Vec：不适合放在主分析流程里，适合放统计页作为扩展模型说明
- KMeans：适合统计页展示“聚类画像”，不适合作为诊断结论
- CRF 模型评估：适合统计页展示，不需要在主界面占过多位置

## 6. 当前项目定位

本项目适合作为课程展示型 NLP 系统，重点是：

```text
超声报告文本 -> 分词 -> 关键词 -> 规则抽取 -> 实体识别 -> 关系抽取 -> 规范化 -> 模板摘要
```

机器学习部分作为增强：

```text
TF-IDF 分类、CRF 序列标注、Word2Vec、KMeans 聚类
```

不建议把项目包装成临床诊断系统。
